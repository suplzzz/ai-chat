# AI Chat — Умный ассистент на базе GigaChat

Приложение представляет собой умного AI-ассистента, работающего на базе нейросети **GigaChat**. Разработано в рамках тестового задания на позицию стажёра Android. 

Проект реализован с соблюдением принципов **Clean Architecture** и паттерна **MVVM**. Приложение умеет не только поддерживать контекстный диалог, но и генерировать изображения, автоматически придумывать названия для чатов и работать в офлайн-режиме с локальной историей переписок.

## Реализованный функционал

**Обязательные требования по ТЗ:**
- **Авторизация:** Интеграция с Firebase Authentication (Email/Пароль). Валидация полей, обработка сетевых ошибок и ошибок сокетов. Автологин при повторном входе.
- **Список чатов:** Локальное хранение истории чатов (Room). Подгрузка данных порциями (Paging 3). Поиск по названию чата прямо в БД.
- **Экран чата:** Полноценный мессенджер с AI. Бот помнит контекст. Обработка состояний загрузки. Возможность повторной отправки сообщений при ошибке. Полноценная поддержка разметки Markdown.
- **Навигация:** Slide-in меню (ModalNavigationDrawer) для перехода между разделами.
- **Профиль:** Отображение данных из Firestore. Редактирование профиля (имя, телефон). Загрузка аватарок. Проверка реального баланса токенов GigaChat. Настройка темы оформления.

**Дополнительные фичи (Опционалки из ТЗ и сверх того):**
- **Генерация изображений:** Бот умеет рисовать по запросу, картинки кэшируются локально.
- **Галерея:** Отдельный экран с Grid-сеткой всех сгенерированных изображений из всех чатов.
- **Авто-нейминг чатов:** При создании нового диалога GigaChat автоматически анализирует первое сообщение пользователя и придумывает для чата короткое осмысленное название.
- **Продвинутый Share Sheet:** Возможность поделиться ответом бота. Если бот сгенерировал картинку, она отправляется в мессенджеры (Telegram, WhatsApp) вместе с текстом.
- **Edge-to-Edge & Modern UI:** Интерфейс переверстан в консистентный "Pill-shape" (округлый) стиль. Системные бары динамически подстраиваются под тему.

## Технологический стек
* **Язык:** Kotlin
* **UI:** Jetpack Compose
* **Архитектура:** Clean Architecture, MVVM
* **Асинхронность:** Coroutines, Flow
* **Инъекция зависимостей:** Hilt
* **Сеть:** Retrofit 2, OkHttp 3, kotlinx.serialization
* **Локальная БД:** Room, Paging 3
* **Облако:** Firebase Auth, Firebase Firestore, Yandex Cloud S3 (хранение аватарок)
* **Медиа:** Coil
* **Хранилище настроек:** Preferences DataStore

## Важные архитектурные решения

1. **Yandex Cloud S3 вместо Firebase Storage:** В связи с недоступностью Firebase Storage для создания новых бакетов в текущем регионе, архитектурно было принято решение абстрагировать облачное хранилище файлов (аватарки) и использовать Yandex S3.
2. **Маршрутизация GigaChat Моделей:** Выбран осознанный подход к распределению токенов. Для генерации ответов и изображений используется `GigaChat-Pro` (как более способная модель), в то время как парсинг и сбор статистики баланса адаптированы под все семейства моделей.

## Запуск проекта

Для сборки проекта вам потребуются API ключи для GigaChat и S3.
1. Склонируйте репозиторий.
2. Поместите ваш файл `google-services.json` в папку `app/`.
3. В корне проекта в файле `keystore.properties` добавьте следующие переменные (подставьте свои значения):

```properties

GIGACHAT_AUTH_KEY="ваш_base64_ключ_от_gigachat"

S3_ACCESS_KEY="ваш_yandex_s3_access_key"

S3_SECRET_KEY="ваш_yandex_s3_secret_key"

S3_ENDPOINT="https://s3.yandexcloud.net"

S3_BUCKET_NAME="имя_вашего_бакета"

```

4. Соберите проект через Android Studio (Min SDK: 26).
